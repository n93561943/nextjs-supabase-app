## 1. 문제 (Problem)

모임을 주최할 때 카카오톡, 엑셀, 계좌이체 앱을 오가며 처리해야 하는 비효율

- **공지/일정 안내**: 단톡방에서 공지가 묻혀 참여자가 놓치는 경우 잦음
- **참여 확인**: 수동 카운팅으로 인원 파악에 실수 발생
- **정산 처리**: 1/N 계산 후 납부 여부를 일일이 확인해야 하는 번거로움
- **참여 신청**: 참여자가 별도 앱 설치 또는 회원가입을 해야 하는 진입 장벽

### 기존 대안

| 도구            | 한계                                    |
| --------------- | --------------------------------------- |
| 카카오톡 단톡방 | 공지가 대화에 묻힘, 참여 현황 파악 불가 |
| 엑셀/구글 시트  | 공유 번거로움, 실시간 반영 어려움       |
| 토스/카카오페이 | 정산만 가능, 참여 관리 기능 없음        |

## 2. 고객 세그먼트 (Customer Segments)

### 주요 고객: 주최자 (Host)

- 수영, 헬스, 러닝, 등산 등 소규모 정기 모임 운영자
- 친목 모임, 스터디, 동호회 등 비정기 이벤트 주최자
- 엑셀/카톡으로 참여자 관리 중인 5~30인 규모 모임 운영자
- 모바일 환경에서 빠르게 처리하고 싶은 개인 주최자

### 보조 고객: 참여자 (Participant)

- 초대 링크를 받아 접속하는 모임 참여자
- 앱 설치나 회원가입 없이 간편하게 신청하고 싶은 일반인

### 얼리 어답터

- 정기적으로 소규모 모임(10인 내외)을 운영하며 정산까지 직접 처리하는 주최자

---

## 3. 가치 제안 (Unique Value Proposition)

> **링크 하나로 모임을 완성한다**
> 주최자는 5분 안에 이벤트를 열고, 참여자는 설치 없이 링크로 신청한다.

- 주최자: 참여 관리 · 공지 · 정산을 하나의 화면에서 처리
- 참여자: 회원가입 없이 링크만으로 신청 → 공지 확인 → 정산 납부 가능
- 자동 1/N 계산으로 정산 실수 제거

---

## 4. 해결책 (Solution)

| #   | 기능                                         | 효과                        |
| --- | -------------------------------------------- | --------------------------- |
| 1   | 이벤트 생성 + UUID 초대 링크 자동 발급       | 모임 개설 5분 이내          |
| 2   | 비회원 참여 신청 (이름 + 연락처)             | 참여자 진입 장벽 제거       |
| 3   | 참여자 승인/거절 + 상태별 목록 조회          | 수동 카운팅 제거            |
| 4   | 공지 작성 · 고정 · 열람 (링크 페이지)        | 공지 누락 방지              |
| 5   | 총액 입력 → 1인당 금액 자동 계산 + 납부 현황 | 정산 오류 및 추적 부담 제거 |

---

## 5. 경쟁우위 (Unfair Advantage)

- **비회원 참여 UX**: 참여자가 앱 설치·회원가입 없이 링크 하나로 모든 것을 처리 — 경쟁 서비스 대비 마찰 최소화
- **모든 기능 통합**: 참여 관리 + 공지 + 정산을 단일 서비스에서 제공 (경쟁 도구는 각각 별도)
- **주최자 온보딩 5분**: 복잡한 설정 없이 이벤트 생성 즉시 공유 가능
- **모바일 최우선 설계**: 주최자/참여자 모두 스마트폰에서 즉시 사용 가능

---

## 6. 수익 흐름 (Revenue Stream)

> MVP 단계에서는 수익 모델 없이 사용자 검증에 집중

### MVP 이후 검토 모델

| 모델          | 설명                                                     |
| ------------- | -------------------------------------------------------- |
| 프리미엄 플랜 | 이벤트 수 제한 해제, 참여자 수 상한 확대, 통계 기능 제공 |
| 정산 수수료   | 외부 결제 연동 시 건당 소액 수수료 (MVP 이후)            |
| 팀/단체 플랜  | 동호회·팀 단위 구독 요금제                               |

## 7. 비용 구조 (Cost Structure)

### 인프라 비용 (초기 거의 무료)

| 항목     | 내용                                                 |
| -------- | ---------------------------------------------------- |
| Vercel   | 무료 플랜으로 시작, 트래픽 증가 시 Pro 플랜 전환     |
| Supabase | 무료 플랜 (500MB DB, 2GB 스토리지)으로 MVP 운영 가능 |

### 운영 비용

- 개발 인건비 (1인 개발 기준 MVP)
- 도메인 비용 (연 ~15,000원)
- 사용자 증가 시 Supabase Pro 전환 비용 ($25/월~)

## 8. 주요 지표 (Key Metrics)

### 활성화 지표

| 지표                         | 목표 (MVP 기준)                     |
| ---------------------------- | ----------------------------------- |
| 주최자 이벤트 생성 완료율    | 가입 후 첫 이벤트 생성까지 5분 이내 |
| 초대 링크 → 참여 신청 전환율 | 링크 클릭 대비 신청 완료 ≥ 60%      |

### 리텐션 지표

| 지표                                | 목표  |
| ----------------------------------- | ----- |
| 월 2회 이상 이벤트 생성 주최자 비율 | ≥ 30% |
| 정산 기능 사용 이벤트 비율          | ≥ 50% |

### 건강 지표

- 이벤트당 평균 참여자 수
- 정산 납부 확인 완료율
- 주최자 온보딩 완료 소요 시간

## 9. 채널 (Channels)

### 획득 채널

| 채널             | 설명                                            |
| ---------------- | ----------------------------------------------- |
| 초대 링크 바이럴 | 참여자가 링크를 받으며 서비스를 자연스럽게 인지 |
| 커뮤니티 공유    | 동호회 카페, 오픈채팅방, 인스타그램 등          |
| SEO              | "모임 정산", "참여자 관리 앱" 등 검색 유입      |

### 유지 채널

- 서비스 내 공지/이벤트 알림
- 이메일 (가입 확인, 이벤트 상태 변경 알림 — MVP 이후)
